# AGENTS.md

## Repository Directory Guide
This file is intentionally short and serves as the top-level directory/navigation document.

## Core Entry Points
- `ARCHITECTURE.md`: system design, module boundaries, and runtime model.
- `docs/CICD.md`: CI/CD workflow, deploy server settings, and operational rules.
- `docs/design-docs/`: design principles and governance docs.
- `docs/exec-plans/`: active/completed execution plans and tech debt tracker.
- `docs/product-specs/`: user-facing behavior specs and acceptance rules.
- `docs/generated/`: generated artifacts (for example DB schema snapshots).
- `docs/references/`: curated technical references.
