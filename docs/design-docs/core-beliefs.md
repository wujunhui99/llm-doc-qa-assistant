# Core Beliefs

1. Harness first, prompt second.
2. Deterministic systems own control flow; LLM owns reasoning.
3. Every turn is observable and replayable.
4. Tool boundaries must be explicit and least-privileged.
5. Quality must be measured through eval loops, not intuition.
6. Grounded answers with citations are default for doc QA.
7. Reliability and fallback paths are part of product behavior.
8. Architecture docs are the source of truth for team alignment.
